Common Sprint Deliverables

  • Description of functionality completed

As of Sprint 1, our team has focused on building the project foundation. We have:

  • Planned the core architecture for the diary system.
  • Set up the development environment using XAMPP (Apache + MySQL + PHP).
  • Designed the database structure for users and diary entries.
  • Created initial HTML and CSS mockups for the login, registration, and dashboard pages.
  • Implemented basic user registration and login functionality using PHP and MySQL.
  • Added the ability for a logged-in user to create and view text-based diary entries stored in the database.

This represents the first functional slice of our application, showing that the diary system can securely store entries linked to individual users.

  • User Guide (how to use the current features)

  • Sign Up - Open the Register page and fill in your name, email, and password.
  • Log In - Enter your email and password to access your dashboard.
  • Dashboard - See a simple text area to create your diary entry and choose a mood.
  • Save Entry - Click Save to store the entry in your personal journal.
  • View Past Entries - Entries appear below the form in reverse chronological order.
  • Upload images or short videos, and see motivational quotes on the dashboard.

  • Retrospective

What Went Well:

  • Team communication and role assignment were clear.
  • Successfully connected PHP scripts to MySQL database via XAMPP.
  • First functional slice (login to entry to display) completed.
What Did Not Go Well:
  • Database setup took longer than expected due to connection errors.
  • Coordinating Git merges caused some delays.
  • Time management was tight toward the deadline.
Improvements for Next Sprint:
  • Plan database structure earlier before coding.
  • Assign a member to handle testing and documentation from the start.
  • Use Git branches for individual features to avoid conflicts.

  • Major Pages

    • Implemented
    1. register.php: User registration page for new accounts.
    2. login.php: Login form that authenticates user credentials.
    3. dashboard.php: Main dashboard showing entry form and past entries.
    4. save_entry.php: Backend script that stores entries in the database.
    Planned
    1. insights.php: Displays mood statistics and graphs.
    2. profile.php: User profile management page.

  • Database Tables

    • Our main tables are the Users table and Entries table. The User table will store information about the user, while the entry table will store the diary entries of the user.


  • PHP Functions

    • Implemented
    1. registerUser(): Inserts new user into the users table.
    2. loginUser(): Verifies email & password, starts session.
    3. saveEntry(): Stores diary entry linked to logged-in user.
    4. getEntriesByUser(): Retrieves all entries for user.
    Planned
    1. getMoodStats(): Generates mood summary.
    2. getMotivationalQuote(): Fetches daily quote from API or DB.
